 What does this tool? 

It helps you to find traces of software licenses such as headers, terms or references. The results from this analysis are written to an SPDX document. It is particularily useful to find free and open source licenses such GPL, Apache, MIT and other licenses mentioned inside a source code project.

At present it will recognize over 18 licenses through 147 file extensions and 22 programming languages. For example, this tool will be very accurate in finding the exact type of GPL that is mentioned inside a project.

In addition, the tool provides a licensing score to let you know how well documented is the licensing portion and what can be improved.


 Why an SPDX report? 

Imagine the following case: You are writing software for a company and they ask you to list the third-party resources in detail. Typically, you can try to use a spreasheet and do this by hand or semi-automatically. But the result will always lack the needed details (which GPL version, which files are affected, ...).

SPDX is an open specification made specifically for documenting these details. You can give the other company an SPDX document and they'll be able to understand in detail which licenses (and copyrights) are present in the code.

Furthermore, with this tool you can specify in detail which files you have authored, which files belong to third-party components and which licenses apply to a given file.

After creating a report, right-click on a file to see the menu with the available options.
