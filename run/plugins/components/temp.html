 FAQ

 What's this "compiling" and "packaging"?  

Imagine it this way, you've already used this tool to manually define which files inside your code are original, which ones belong to third-party providers and to which components they belong.

This tool will process the information you generated and create a set folders. For example, a folder with the code that you need to make available to public, a folder with the code that is yours (authored).

It will then create the documentation where needed: a LICENSE file is automatically generated, a text file for third-party components is included, and so on.

In the end you get a zip file that should be as (humanly) complete as possible in regards to licensing documentation, that's your new package with the code.

We use the term "compile" because you start the process and it will only complete if everything is ready. If something is missing, it will let you know (and give tips where to fix). 

A more detailed description of how this is done is written on this wiki page.
